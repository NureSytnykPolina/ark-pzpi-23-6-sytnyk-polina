РОЗРОБКА VISION & SCOPE

Назва проєкту
Українською мовою:
Програмна система для організації та управління сервісом оренди одягу
Англійською мовою:
Clothing Rental Management System
Власна назва проєкту:
Click&Rent

1. Vision
1.1 Аналіз проблеми, яку вирішує програмна система
Сфера оренди одягу активно розвивається у зв’язку з ростом попиту на короткострокове використання речей для заходів, фотосесій та урочистостей. Багато сервісів оренди одягу використовують застарілі або неавтоматизовані методи обліку клієнтів, одягу та оренд, що призводить до помилок, втрати даних та неефективного управління ресурсами.
Програмна система Click&Rent покликана вирішити проблему централізованого управління даними сервісу оренди одягу шляхом автоматизації обліку клієнтів, одягу, типів одягу та процесу оренди.

1.2 Аналіз аналогів
Серед існуючих аналогів можна виділити онлайн-платформи оренди одягу та локальні CRM-системи, які дозволяють вести базовий облік замовлень. Проте більшість таких рішень є або надто дорогими, або не адаптованими під потреби невеликих сервісів оренди.
Перевагою системи Click&Rent є простота використання, модульна архітектура та можливість подальшого розширення функціоналу.
Монетизація проєкту може здійснюватися шляхом:
•	підписки на використання системи;
•	надання розширених функцій для адміністраторів;
•	інтеграції платіжних сервісів.

2. Scope
2.1 Опис призначення системи
Програмна система Click&Rent призначена для автоматизації процесів обліку та управління сервісом оренди одягу. Система забезпечує зручну взаємодію між клієнтами та адміністраторами, а також контроль за станом орендованого одягу.

2.2 Опис бізнес-логіки системи
Бізнес-логіка системи полягає у взаємодії клієнтів з доступним одягом та оформленні оренди. Адміністратор системи керує каталогом одягу, типами одягу, клієнтами та орендами. Кожна оренда фіксується з датою початку та завершення, а також з відповідною вартістю.

3. Функціональні вимоги
3.1 Опис функціональності серверної частини та IoT-клієнта
Серверна частина системи реалізує:
•	зберігання та обробку даних;
•	REST API для взаємодії з клієнтами;
•	управління клієнтами, одягом, типами одягу та орендами;
•	реалізацію CRUD-операцій.

3.2 Опис функціональності веб-клієнта та мобільного клієнта
Веб-клієнт дозволяє:
•	перегляд каталогу одягу;
•	оформлення оренди;
•	перегляд історії оренд.
Мобільний клієнт передбачає аналогічну функціональність з адаптацією під мобільні пристрої.

4. Обмеження та зацікавлені сторони
4.3 Опис програмних засобів та програмних інструментів
У проєкті використовуються наступні технології:
•	Платформа: .NET 8
•	Фреймворк: ASP.NET Core Web API
•	Мова програмування: C#
•	База даних: SQL Server Express
•	ORM: Entity Framework Core
•	Документація API: Swagger
•	Середовище розробки Visual Studio;
•	Система керування версіями: GitHub
Зацікавлені сторони проєкту:
•	власники сервісу оренди одягу;
•	адміністратори системи;
•	клієнти сервісу.
