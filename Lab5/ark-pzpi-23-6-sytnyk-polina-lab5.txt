МЕТА ТА ЗАВДАННЯ РОБОТИ
Мета роботи
Метою лабораторної роботи є набуття практичних навичок розгортання програмної системи, перевірки її працездатності та демонстрації взаємодії між серверною частиною, бізнес-логікою та IoT-клієнтом на прикладі системи оренди одягу Click&Rent.
Завдання роботи
У процесі виконання лабораторної роботи необхідно:
•	розгорнути серверну частину програмної системи;
•	перевірити коректність роботи бізнес-логіки;
•	продемонструвати функції адміністрування;
•	продемонструвати роботу IoT-клієнта;
•	показати взаємодію IoT-клієнта та серверної частини;
•	створити відеозапис демонстрації функціональності системи;
•	підготувати та оформити звіт.

2. ОПИС РОЗГОРТАННЯ ПРОГРАМНОЇ СИСТЕМИ
Програмна система Click&Rent була розгорнута у локальному середовищі з використанням платформи .NET 8 та фреймворку ASP.NET Core Web API.
Для зберігання даних використовується реляційна база даних SQL Server, підключення до якої здійснюється за допомогою ORM Entity Framework Core. Параметри підключення визначені у конфігураційному файлі appsettings.json.
Після запуску серверної частини система стає доступною за локальною адресою https://localhost:7068, а для тестування REST API використовується інтерфейс Swagger UI.

3. ДЕМОНСТРАЦІЯ ФУНКЦІОНАЛЬНОСТІ СИСТЕМИ
3.1 Демонстрація бізнес-логіки
Основним бізнес-процесом системи є оформлення оренди одягу. Бізнес-логіка реалізована у сервісному шарі та включає:
•	перевірку коректності дат оренди;
•	перевірку існування клієнта та одягу;
•	перевірку доступності одягу;
•	автоматичний розрахунок вартості оренди.
Перевірка бізнес-логіки здійснюється через ендпоінт:
POST /api/rents-business/create-with-check
У разі коректних даних сервер повертає статус 200 OK, а у разі помилкових даних — відповідне повідомлення про помилку.

3.2 Демонстрація функцій адміністрування
Для адміністратора реалізовано окремі функції керування системою. Зокрема, реалізовано отримання статистики системи, що дозволяє переглядати загальну кількість клієнтів, одягу та оренд.
Перевірка адміністративної функціональності здійснюється через ендпоінт:
GET /api/admin/stats

3.3 Демонстрація роботи IoT-клієнта
У рамках лабораторної роботи було використано IoT-клієнт, реалізований у вигляді Java-консольного застосунку WearSensorSimulator.
IoT-клієнт імітує роботу сенсорів одягу та реалізує:
•	генерацію значень температури;
•	генерацію значень вологості;
•	періодичну передачу даних на сервер через HTTP POST запити у форматі JSON.
Передача даних здійснюється на ендпоінт:
POST /api/measurements
У разі успішної передачі сервер повертає статус 200 OK, що підтверджує коректну взаємодію IoT-клієнта з серверною частиною системи.

3.4 Взаємодія IoT-клієнта та серверної частини
Взаємодія між IoT-клієнтом та сервером реалізована за допомогою REST API. Сервер приймає дані, обробляє їх та підтверджує успішне отримання. Таким чином демонструється інтеграція IoT-компонента у загальну архітектуру програмної системи Click&Rent.

4. РЕЗУЛЬТАТИ ТЕСТУВАННЯ
У процесі тестування було підтверджено:
